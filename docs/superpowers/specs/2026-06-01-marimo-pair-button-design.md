# marimo Pair Button — Design

**Date:** 2026-06-01
**Status:** ready
**Repo:** marimo-pycharm (PyCharm/IntelliJ plugin, Kotlin)

## Goal

Add a "marimo pair" control to the PyCharm plugin that, in one click, opens a
PyCharm terminal and **automatically starts** a chosen AI coding harness
(Claude, Codex, or `pi`) already loaded with the marimo-pair prompt for the
currently open notebook.

**Definition of done:** From an open marimo notebook, the user clicks the pair
control, picks a harness, and a terminal tab opens running that harness with the
pair prompt — no copy-paste, no manual command entry.

## Key decision: execute, don't copy-paste

marimo's frontend `PairWithAgentModal`
(`~/Projects/marimo/frontend/src/components/editor/actions/pair-with-agent-modal.tsx`)
renders command strings for the user to **copy and paste** into their own
terminal. This plugin instead **executes** the equivalent command in an embedded
PyCharm terminal. The modal is the reference for *what command to run*; the
plugin's job is to *run it automatically*.

The prompt itself is generated by the marimo CLI, not templated by the plugin:

```
marimo pair prompt --url '<url>' [--claude|--codex]
```

(Source: `~/Projects/marimo/marimo/_cli/pair/commands.py`.) Shelling out keeps us
in sync with upstream automatically.

## Harness commands

`<M>` = marimo CLI prefix (see "Resolving the marimo CLI"). `<url>` = running
server URL. The embedded server runs with `--no-token`, so **no token handling**.

| Harness | Terminal command |
|---|---|
| Claude | `claude "$(<M> pair prompt --url '<url>' --claude)"` |
| Codex  | `codex "$(<M> pair prompt --url '<url>' --codex)"` |
| pi     | `pi "$(<M> pair prompt --url '<url>')"` |

`pi` uses no agent flag — upstream `marimo pair prompt` has none, and the echoed
prompt text is identical regardless of flag (the `--claude`/`--codex` flags only
emit a skill-presence warning on stderr). All three start the harness
**interactively** with the prompt as the first message.

## UI surfaces (A + C, one shared action)

A single `MarimoPairAction` is the brain. On invocation it shows a JBPopup
listing **Claude / Codex / pi**; the chosen harness is launched. Two surfaces
invoke it so the user can compare and keep one:

- **A — in-editor toolbar:** an `ActionToolbar` strip pinned at
  `BorderLayout.NORTH` of `MarimoNotebookEditor`, above the JCEF browser. Always
  visible while a notebook is open.
- **C — editor tab context menu + Tools menu:** the same action contributed to
  `EditorTabPopupMenu` and the Tools menu, enabled only when the active file is a
  marimo notebook.

Note: the marimo view is a custom JCEF `FileEditor`, not a text editor, so the
IDE's normal floating editor toolbar does not apply; the editor's own toolbar
strip (A) is the in-editor surface and the tab context menu (C) is the
lightweight one.

## Resolving the marimo CLI + URL

- **URL:** from `MarimoServerService.urlFor(file)` — ensures the server is up and
  returns the ready URL. The action runs after the notebook editor is open, so
  the server is typically already running.
- **marimo CLI prefix:** reuse the launcher that started the server so the
  terminal uses the *same* marimo. Add `marimoCliPrefix(): List<String>` to
  `MarimoLauncher`:
  - `UvLauncher` → `uv run --with marimo marimo`
  - `SdkLauncher` → `<python> -m marimo`

## Opening the terminal

Use the bundled Terminal plugin:

```kotlin
TerminalToolWindowManager.getInstance(project)
    .createLocalShellWidget(workDir, "marimo pair")
    .executeCommand(cmd)
```

`workDir` = the notebook's parent directory. Add
`<depends>org.jetbrains.plugins.terminal</depends>` to `plugin.xml`.

## Error handling

- No applicable launcher / server fails to start → balloon notification reusing
  the editor's existing error phrasing.
- Terminal plugin unavailable → notification including the command text so the
  user can run it manually.

## Components / files

- `pair/MarimoHarness.kt` — enum (id, label, command builder).
- `pair/MarimoPairLauncher.kt` — builds the command, opens the terminal.
- `pair/MarimoPairAction.kt` — `AnAction`, harness popup, enablement.
- `MarimoLauncher.marimoCliPrefix()` — added to interface + both impls.
- `MarimoNotebookEditor` — add the toolbar strip (A).
- `plugin.xml` — action registration (C), terminal dependency.
- Tests: command-string building per harness (pure, like existing
  `UvLauncherCommandTest`).

## Verification

1. Build/run the plugin (`runIde`) with a marimo notebook open.
2. Confirm the in-editor toolbar (A) shows a "Pair" button and the editor tab
   right-click menu (C) shows the same action.
3. Click it → popup lists Claude/Codex/pi.
4. Pick `pi` (installed locally) → a terminal tab opens and `pi` starts with the
   pair prompt visible; the agent can reach the running notebook URL.
5. Repeat for Claude and Codex; verify the exact command matches the table above.
6. Unit tests for command building pass.
